# Agent Orchestration Patterns（サブエージェント・オーケストレーション設計）

複数のサブエージェントで1つのゴールを達成する際の品質は、モデルの地力以上に**オーケストレーションの設計**に左右されます。本プラグインは、その設計を再現するための原則・パターン・起動前チェックリストをまとめた Claude Code 用スキル（`designing-agent-workflows`）です。

## 使い方

スラッシュコマンド `/fab` のあとに、やりたい作業を書くだけ。

```
/fab この監査を手分けして設計・実行して
```

スキル名で直接呼ぶこともできます（例：`designing-agent-workflows を使って、この監査を設計して`）。エージェントを起動する前に読むのが最も効果的です。

## 収録内容

- 3つの思考モデル（並走か同期か／独立した証拠で検証／権威ある受け渡し）
- 6つの失敗モードと回避法
- 起動前チェックリスト
- 7ジャンルの実践パターン（レビュー／監査・研究・設計・変換・移行・検証専用・法務検証）

> インストール手順・設計思想の詳細は、リポジトリのルート [README](../../README.md) を参照してください。
