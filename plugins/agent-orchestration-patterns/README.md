# Agent Orchestration Patterns（サブエージェント・オーケストレーション設計）

複数のサブエージェントで1つのゴールを達成する際の品質は、モデルの地力以上に**オーケストレーションの設計**に左右されます。このプラグインは、その設計を再現するための原則・パターン・起動前チェックリストをまとめた Claude Code 用スキルです。設計不備による品質の下振れを防ぎ、安定して高品質なマルチエージェント成果を引き出します。

---

## 何が手に入るか

スキル `designing-agent-workflows` 1本（`SKILL.md` + `PATTERNS.md`）：

- **3つの思考モデル** — 並走か同期か／独立した証拠で検証／権威ある受け渡し
- **6つの失敗モードと回避法** — 独立作業の直列化／単一票検証／網羅ゴールの単発パス／サイレントな件数上限／曖昧な受け渡し（再探索）／実装者を信用した検証
- **天井を上げるテクニック** — 自己完結プロンプト、戻り値形式の固定、共有コントラクト、自己申告フィールド、棄却ログ保全、2トラック検証 ほか
- **起動前チェックリスト** — 起動前に1分で品質を担保
- **7ジャンルの実践パターン**（レビュー／監査・研究・設計・変換・移行・検証専用・法務検証）— 標準のサブエージェント（Task/Agent）ツール向けに記述
- **実行済みケーススタディ** — 約30体規模のエージェント・数十万〜百万トークン規模の研究パイプライン（数十件規模の主張を収集→大多数を検証採用・少数を棄却）
- **付録** — 決定論的ワークフロー・スクリプティングツールを持つ環境向けの上級コード例

## 必要環境

- **Claude Code**（プラグイン対応バージョン）
- **特別なツールは不要。** 本体はClaude Codeに標準搭載のサブエージェント（Task/Agent）ツールで実践できるよう書かれています。
- 決定論的ワークフロー・スクリプティングツールがある環境では、付録の上級パターンも利用できます（任意）。

## インストール

```
# 1) マーケットプレイスを登録（配布先のGitリポジトリを指定）
/plugin marketplace add <owner>/<repo>

# 2) プラグインをインストール
/plugin install agent-orchestration-patterns@aop-marketplace

# 3) 確認
/plugin list
```

ローカルで検証する場合（開発者向け）は、`/plugin marketplace add` にこのリポジトリのローカルパスを指定してください。

## 使い方

インストール後、マルチエージェントのオーケストレーション（監査・レビュー・調査・設計・移行・要約など）を始めると、スキルが自動的に候補に挙がります。確実に適用したいときは次のように依頼します。

```
designing-agent-workflows を使って、この監査を設計して
```

このスキルは「エージェントを起動する前に」読むのが最も効果的です。
